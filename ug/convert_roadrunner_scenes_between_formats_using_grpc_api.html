使用 gRPC API 在格式之间转换场景

此示例演示如何从一种文件格式导入 RoadRunner 场景，并将这些场景导出为其他格式。在 此示例，您将 ASAM OpenDRIVE 文件导入场景，将它们保存到项目中，然后使用命令行操作将场景导出到 Filmbox FBX 文件。

RoadRunner gRPC API 的工作原理

RoadRunner 提供导入和导出场景和场景的 API 服务 编程。此 API 是使用开源、与语言无关的 gRPC 框架构建的，该框架使你能够对 RoadRunner 服务器进行远程过程调用 （RPC），以编程方式控制应用程序。有关更多背景信息，请参阅使用 gRPC API 以编程方式控制 RoadRunner。®
您可以将 RoadRunner API 服务编译为 gRPC 支持的任何编程语言，并编写客户端以使用该语言远程控制 RoadRunner。RoadRunner 还提供了此 API 的预编译版本作为命令行工具。此示例使用此预编译的帮助程序命令来执行这些操作。
注意
此示例主要使用 Windows 命令和文件路径来调用 API，但此 API 也适用于 Linux。

此脚本执行以下操作：
  1. 打开 RoadRunner 到一个项目并启动 RoadRunner API 服务器。
  2. 将文件夹中包含的每个 ASAM OpenDRIVE 文件加载到新场景中。
  3. 使用默认设置将每个场景导出为 Filmbox FBX 格式。对于每个导出的场景，RoadRunner 会将 FBX 文件和关联的纹理图像文件导出到项目文件夹下的子文件夹中。Exports
  4. 将场景保存到文件夹。每个场景都有 与 ASAM OpenDRIVE 文件同名，但带有后缀和 扩展名而不是 .Scenes_OpenDRIVE.rrscene.xodr
  5. 退出 RoadRunner 并关闭 RoadRunner 服务器。
