使用 gRPC API 以编程方式控制 RoadRunner

RoadRunner 提供了一个 API，使您能够以编程方式控制 RoadRunner UI。例如，使用此 API，您可以：
  • 创建、加载和保存 RoadRunner 场景、场景和项目。
  • 将 ASAM OpenDRIVE 文件导入场景。®
  • 将场景和场景导出为 RoadRunner 支持的文件格式之一。
RoadRunner使您能够以各种编程语言编译API的版本 并用您选择的语言呼叫他们。或者，您可以使用预编译版本 的 API，使您能够从命令行控制 RoadRunner。RoadRunner API 的工作原理
RoadRunner API 是使用开源 gRPC 框架构建的。此框架使用客户端-服务器体系结构，其中 客户端应用程序使用一组远程远程控制服务器应用程序 过程调用 （RPC） 方法。在RoadRunner中：®
  • 您本地安装的RoadRunner版本是服务器应用程序。
  • RoadRunner API 提供了用于远程控制 RoadRunner 的 RPC 方法。
  • 为调用 RPC 方法而编写的程序是客户端应用程序。 gRPC 框架与语言无关，与平台无关。你可以写 以 gRPC 支持的任何平台和语言调用 RoadRunner API 的客户端。有关 gRPC 支持的语言和平台的详细信息，请参阅 gRPC 文档。

连接到RoadRunner API服务器

要使用RoadRunner API，您必须先与RoadRunner API服务器建立网络连接。此服务器是本地 RoadRunner 安装的一部分，并在您打开项目时开始运行。
要以编程方式打开RoadRunner并启动API服务器，请从本地RoadRunner安装中调用AppRoadRunner可执行文件。 此可执行文件包含启用的命令行选项 您要指定：
  • RoadRunner 打开的项目
  • 运行 RoadRunner API 服务器的 IP 网络端口
此命令行代码显示了如何从 Windows 上的默认安装位置打开 RoadRunner。RoadRunner 打开一个位于 IP 的项目 网口 。®C:\RR\MyProject54321
cd "C:\Program Files\RoadRunner R2023a\bin\win64"
AppRoadRunner --projectPath C:\RR\MyProject --apiPort 54321
RoadRunner 的“输出”窗格显示运行 RoadRunner API 服务器的端口。

使用各种编程语言的RoadRunner API

为了提高使用 RoadRunner API 的灵活性，您可以将 API 编译为 gRPC 支持的语言，然后使用该语言编写客户端应用程序以编程方式控制 RoadRunner。
编译 RoadRunner API
要用所需的编程语言编译RoadRunner API，您必须首先复制 将 API 定义为可写文件夹的 protobuf 文件。这些文件位于 在您当地的 RoadRunner 安装中。然后，您可以使用 protobuf 编译器以及 用于所需编程语言的 gRPC 插件，用于编译 特定于语言的版本或绑定的 RoadRunner API。例如，此图显示了 Python 和 C++ 绑定的生成。
创建 RoadRunner API 客户端
您编写的用于以编程方式控制 RoadRunner 的客户端通常包含执行以下步骤的代码：
  1. 从已编译的绑定导入 gRPC 代码。
  2. 建立与RoadRunner API服务器的本地网络连接。
  3. 使用导入的 gRPC 代码创建 RoadRunner API 对象。此对象称为存根。
  4. 从此存根调用 RPC 方法以通过本地网络控制 RoadRunner。
这个简单的 Python 客户端显示了调用该方法的示例。LoadScene
import grpc
from mathworks.roadrunner import roadrunner_service_messages_pb2
from mathworks.roadrunner import roadrunner_service_pb2_grpc
with grpc.insecure_channel("localhost:54321") as channel:
api = roadrunner_service_pb2_grpc.RoadRunnerServiceStub(channel)
loadSceneRequest = roadrunner_service_messages_pb2.LoadSceneRequest()
loadSceneRequest.file_path = "FourWaySignal"
api.LoadScene(loadSceneRequest)
导入已编译的 gRPC 后 Python 绑定，此客户端通过 gRPC 通道建立连接，并为 RoadRunner 服务 API 创建存根。API 的 gRPC 通道使用不安全的通道凭据，并且没有 加密或身份验证。由于RoadRunner通常在本地计算机上运行并且不连接到外部 网络，安全风险低。然后，客户端从 API 存根调用，该存根从当前打开的项目加载预生成场景。LoadSceneFourWaySignal
您可以让多个客户端同时调用 RPC 方法，只要它们是 连接到同一网络端口上的RoadRunner。例如，在此图中，两个 Python 客户端和C++客户端正在调用 网口 。LoadScene54321
