在控制台模式下以编程方式控制 RoadRunner

RoadRunner 提供用于执行常见工作流任务的编程接口，例如 打开、关闭和保存场景和项目以及导入和导出场景。您可以 使用 MATLAB 函数或 gRPC API 来执行这些常见的工作流任务。MATLAB 函数通过 MATLAB 对象进行操作，该对象打开并设置主 RoadRunner 用户界面，以便您可以通过编程方式对其进行控制。这些 MATLAB 函数需要自动驾驶工具箱™许可证。gRPC API 使用远程过程调用 （RPC） 方法，该方法在一组协议中定义 缓冲区 （protobuf） 架构文件，使您能够以编程方式远程控制 RoadRunner。使用 gRPC 框架，可以将这些文件编译为 C++、Python 或 gRPC 支持的其他编程语言。然后，可以编写调用这些方法的客户端应用程序 以您选择的语言控制RoadRunner。
使用 MATLAB 函数和 gRPC API，您可以在控制台模式下启动 RoadRunner，也允许在非图形环境中从终端运行 RoadRunner。这使得 分布式工作流程，例如，从远程服务器运行RoadRunner，并使您能够以更大的规模使用RoadRunner。RoadRunner 通常用作可能需要的模拟器中管道的一部分 将场景导入 RoadRunner 并将其导出为与其他模拟器兼容的格式。在控制台模式下使用 RoadRunner 可提高与此类持续集成 （CI） 的集成 系统。它还通过减少加载所需的时间来提高应用程序性能 RoadRunner 中的场景，因为您不需要加载和显示场景图形。
与交互式RoadRunner一样，RoadRunner控制台可以接收从编程接口发送的API命令。您可以 通过编译原型文件、使用应用程序或 RoadRunner 的 MATLAB 函数来编写您自己的客户端。如果您使用的是 gRPC API，请使用 AppRoadRunner和参数在非图形环境中启动 RoadRunner。RoadRunner 提供了一个预编译的辅助命令， 这使您能够从命令行调用 RoadRunner RPC 方法。此帮助程序命令位于同一 文件夹作为可执行文件。如果您使用的是 MATLAB，请使用带有 NoDisplay属性的 roadrunner对象在非图形环境中启动 RoadRunner。CmdRoadRunnerApi--nodisplayCmdRoadRunnerApiAppRoadRunner
