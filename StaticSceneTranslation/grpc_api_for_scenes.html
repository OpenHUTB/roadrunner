gRPC场景和高清地图的 API

RoadRunner 提供了一个用于执行常见工作流任务的 API，例如 打开、关闭和保存场景和项目以及导入和导出 场景。此 API 使用远程过程调用 （RPC） 方法，该方法在一组 协议缓冲区 （protobuf） 架构文件，使您能够以编程方式远程控制 RoadRunner。使用 gRPC 框架，可以将这些文件编译为 C++、Python 或 gRPC 支持的其他编程语言。然后，可以编写调用这些方法的客户端应用程序 以您选择的语言控制RoadRunner。

功能

AppRoadRunner 使用 gRPC 从命令行启动 RoadRunner 应用程序

远程过程调用方法

项目
NewProject使用 gRPC 创建新的 RoadRunner 项目
LoadProject使用 gRPC 加载 RoadRunner 项目
SaveProject使用 gRPC 保存 RoadRunner 项目
现场
NewScene使用 gRPC 创建新的路跑者场景
LoadScene使用 gRPC 加载 RoadRunner 场景
SaveScene使用 gRPC 保存路跑者场景
导出、导入和退出
Export使用 gRPC 导出路跑者场景或方案
Import使用 gRPC 将文件导入到路跑者场景或方案中
Exit使用 gRPC 退出 RoadRunner 应用程序

协议缓冲区架构文件

roadrunner_service.proto RoadRunner service protobuf schema
roadrunner_service_messages.proto RoadRunner Service messages protobuf schema
import_settings.proto 导入设置 protobuf 架构
export_settings.proto 导出设置 protobuf 架构
geometry.proto 场景几何原型方案

主题

开始
使用 gRPC API 以编程方式控制 RoadRunner API是使用 gRPC 框架构建的。使用此 API，您可以管理 RoadRunner 项目、场景和场景，导入或导出场景和场景， 并以您选择的语言执行其他编程任务。
使用接口
在控制台模式下以编程方式控制RoadRunner 在非图形环境中使用终端启动RoadRunner。
使用 gRPC API在格式之间转换场景 以一种格式导入 RoadRunner 场景数据，并使用 命令行 gRPC 客户端。
使用 gRPC API导出多个场景 使用命令行将项目中的所有 RoadRunner 场景导出为支持的格式 gRPC 客户端。
构建 API 客户端
编译 RoadRunner gRPC API 的协议缓冲区将定义 RoadRunner 服务 API的协议缓冲区 （protobuf） 文件编译为所需的编程语言。
创建用于以编程方式控制RoadRunner的gRPC Python客户端 创建客户端gRPCAPI，用于在Python中使用RoadRunner。
创建用于以编程方式控制 RoadRunner 的 gRPC C++客户端 创建客户端 gRPC API，以便在 C++中使用 RoadRunner。
